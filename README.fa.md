# ارسال پیامک بلند | Long SMS Sender

[🇺🇸 English](README.md) | [🇮🇷 فارسی](README.fa.md)

![ارسال پیامک بلند](https://github.com/mostafaafrouzi/long-sms-sender/raw/main/long-sms-sender-fa.png)

یک برنامهٔ اندروید است که به شما امکان می‌دهد متن‌های **خیلی بلند** را به‌صورت **پیامک چندبخشی واقعی** بفرستید و **تا حد ممکن از تبدیل ناخواسته به MMS** جلوگیری کنید.

---

## 📱 این برنامه چه می‌کند؟

با «ارسال پیامک بلند» می‌توانید:

- متن **نامحدود** را به‌صورت **چند پیامک پشت‌سرهم** (SMS) بفرستید، نه MMS.
- از **تبدیل خودکار به MMS** در بسیاری از اپ‌های پیش‌فرض کمتر آسیب ببینید.
- گیرنده را از **مخاطبین** (جستجو، چند انتخابی) بگیرید یا **چند شماره را خط‌به‌خط** دستی وارد کنید.
- **گروه گیرنده** بسازید، ذخیره کنید، بعداً بارگذاری یا حذف کنید.
- روی گوشی **دو سیم‌کارته**، **سیم ارسال** را انتخاب کنید.
- **زمان‌بندی ارسال** داشته باشید؛ در صورت نیاز **بهینه‌سازی باتری**؛ مدیریت از **نوار بالا** (آیکون ساعت)—لغو، ارسال فوری، زمان جدید.
- **پیام آماده (قالب)** ذخیره و کنار متن استفاده کنید.
- از **اشتراک‌گذاری** یا لینک‌های `sms` / `smsto` متن و گیرنده را به برنامه بفرستید.
- برای چند گیرنده **تأیید** بگیرید؛ برای ارسال طولانی **توقف**، **ادامه** یا **لغو صف** داشته باشید.
- **شمارنده بخش پیامک**، **گزارش ارسال** و **اشتراک گزارش** ببینید.

**رابط:** **فارسی و انگلیسی** با تعویض سریع؛ تم **تاریک / روشن / پیروی از سیستم**؛ **راست‌به‌چپ**؛ چیدمان **موبایل و تبلت**.

## 🛡️ چرا MMS نمی‌شود؟

این برنامه از **`SmsManager.sendMultipartTextMessage()`** استفاده می‌کند:

- ارسال به‌صورت **چندبخشی SMS**  
- **بدون موضوع** و **بدون ضمیمه**  
- با رعایت الگوهایی که کمتر MMS را تحریک کنند  

برخلاف بسیاری از پیام‌رسان‌های پیش‌فرض که متن بلند را MMS می‌کنند، اینجا مسیر **SMS** حفظ می‌شود.

## 🔒 حریم خصوصی

**دادهٔ شخصی جمع‌آوری، ذخیره یا ارسال نمی‌شود.**

- بدون اینترنت؛ **آفلاین**  
- بدون آنالیتیکس و تبلیغ  
- پیامک فقط از **دستگاه شما**  
- مخاطب فقط هنگام **باز کردن لیست مخاطبین** و **محلی**  
- سرور سفارشی برای محتوای شما وجود ندارد؛ متن «درباره» **داخل برنامه** است

با رد کردن مخاطبین یا وضعیت تلفن، معمولاً می‌توان با **شماره دستی** و **سیم پیش‌فرض** کار کرد.

## 🏗️ نحوه ساخت (محلی)

### پیش‌نیازها

- **Android Studio** Hedgehog (2023.1.1) یا جدیدتر (توصیه می‌شود)  
- **JDK** 17 به بالا  
- **Android SDK** 34 (از SDK Manager)  
- **Gradle Wrapper** داخل پروژه (نسخه سازگار 8.x)  

### کلون کردن مخزن

```bash
git clone https://github.com/mostafaafrouzi/long-sms-sender.git
cd long-sms-sender
```

### باز کردن در Android Studio

1. **File → Open**  
2. پوشهٔ `long-sms-sender` را انتخاب کنید  
3. صبر کنید تا Gradle sync تمام شود  

### بیلد دیباگ (بدون نیاز به امضا در Gradle)

```bash
./gradlew assembleDebug
```

یا از منو: **Build → Build Bundle(s) / APK(s) → Build APK(s)**.

### محل فایل‌های APK

```text
دیباگ:   app/build/outputs/apk/debug/app-debug.apk
Release: app/build/outputs/apk/release/app-release-unsigned.apk
```

> **توجه:** اگر `signingConfig` در `build.gradle` تعریف نشده باشد، خروجی release **unsigned** است. در **گیت‌هاب** اگر Secrets تنظیم باشد، workflow APK را **امضا** می‌کند.

## 🔏 امضای Release به‌صورت محلی (اختیاری)

برای `assembleRelease` امضاشده روی کامپیوتر خودتان، keystore اضافه و `signingConfigs` را در `app/build.gradle.kts` وصل کنید (فایل رمز و keystore را **کامیت نکنید**).

**۱. ساخت keystore (یک‌بار):**

```bash
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias
```

**۲. فایل `keystore.properties` در ریشه پروژه (کامیت نشود):**

```properties
storeFile=my-release-key.jks
storePassword=رمز-ذخیره-شما
keyAlias=my-key-alias
keyPassword=رمز-کلید-شما
```

**۳. پس از اتصال به Gradle:**

```bash
./gradlew assembleRelease
```

برای **به‌روزرسانی** روی نسخهٔ کافه‌بازار، از **همان keystore** استفاده کنید.

## 🚀 GitHub Actions: انتشار خودکار

فایل **`.github/workflows/release.yml`** با **پوش کردن تگ** به شکل `v*` اجرا می‌شود.

### خلاصه کارها

- اجرای `./gradlew assembleRelease`  
- اگر **Secrets** تنظیم باشد: decode کردن keystore و **امضا** با `apksigner`  
- ساخت **GitHub Release** و ضمیمه **APK**  

### ساختن و ارسال تگ (مثال: v1.1.0)

```bash
git tag -a v1.1.0 -m "نسخه 1.1.0"
git push origin v1.1.0
```

### Secrets مورد نیاز (برای امضا در CI)

| نام | نقش |
|-----|-----|
| `KEYSTORE_BASE64` | فایل `.jks` به‌صورت **یک خط** base64 |
| `KEYSTORE_PASSWORD` | رمز keystore |
| `KEY_PASSWORD` | رمز کلید |
| `KEY_ALIAS` | نام مستعار کلید |

**نمونهٔ base64 (لینوکس / macOS):**

```bash
# لینوکس
base64 -w 0 my-release-key.jks

# macOS
base64 -i my-release-key.jks
```

مقدار را در **تنظیمات مخزن → Secrets and variables → Actions** قرار دهید.

> برای **جایگزینی** نسخهٔ نصب‌شده روی کافه‌بازار، **همان keystore** نسخهٔ منتشرشده را در Secrets بگذارید.
