# ارسال پیامک بلند | Long SMS Sender

[🇺🇸 English](README.md) | [🇮🇷 فارسی](README.fa.md)

![ارسال پیامک بلند](long-sms-sender.png)

یک برنامه اندروید حرفه‌ای که به کاربران امکان ارسال پیامک‌های بسیار بلند به صورت پیامک چندبخشی واقعی را می‌دهد و **به طور کامل از تبدیل به MMS جلوگیری می‌کند**.

## 📱 این برنامه چه می‌کند؟

ارسال پیامک بلند به شما امکان می‌دهد:
- پیامک‌های **با طول نامحدود** را به صورت **چندبخشی واقعی** ارسال کنید
- از **تبدیل خودکار به MMS** که اپراتورها اغلب انجام می‌دهند جلوگیری کنید
- گیرندگان را از مخاطبین با جستجو و چندانتخابی بگیرید یا شماره را **چند سطره** دستی وارد کنید
- **گروه گیرنده** بسازید، ذخیره کنید و برای ارسال بعدی بارگذاری کنید
- روی گوشی **دو سیم‌کارته**، **سیم پیش‌فرض یا سیم مشخص** را برای ارسال انتخاب کنید
- **زمان‌بندی ارسال** با تأیید؛ در صورت نیاز راهنمای بهینه‌سازی باتری؛ مدیریت از نوار بالا (آیکون ساعت)—لغو، ارسال فوری یا زمان جدید
- **پیام‌های آماده (قالب)** ذخیره و کنار فیلد پیام استفاده کنید
- از طریق **اشتراک‌گذاری** و لینک‌های `sms`/`smsto` متن و گیرنده را به برنامه بفرستید
- به یک یا چند گیرنده با تأیید؛ برای ارسال طولانی **توقف / ادامه / لغو صف**
- شمارشگر بخش‌ها، گزارش ارسال و اشتراک گزارش
- فارسی و انگلیسی با تعویض آنی (برچسب‌ها و تقویم با زبان برنامه هماهنگ می‌مانند)
- تم تاریک/روشن/سیستم، RTL، چیدمان تبلت

## 🛡️ چرا از تبدیل MMS جلوگیری می‌کند؟

این برنامه از **`SmsManager.sendMultipartTextMessage()`** استفاده می‌کند که:
- ✅ پیام‌ها را به صورت **پیامک چندبخشی واقعی** ارسال می‌کند (نه MMS)
- ✅ از **بدون خط موضوع** استفاده می‌کند (پارامتر null)
- ✅ **هیچ ضمیمه‌ای** ندارد
- ✅ از الگوهای ایموجی که باعث تبدیل MMS می‌شوند اجتناب می‌کند
- ✅ اطمینان می‌دهد که پیام‌ها به صورت **پیامک استاندارد** ارسال می‌شوند، صرف نظر از طول

برخلاف برنامه‌های پیام‌رسانی پیش‌فرض که پیام‌های بلند را به MMS تبدیل می‌کنند (که هزینه بیشتری دارد و ممکن است توسط همه اپراتورها پشتیبانی نشود)، این برنامه تضمین می‌کند که پیام‌های شما به صورت SMS ارسال شوند.

## 🔐 توضیح مجوزها

مجوزها **در زمان نیاز** و با توضیح درخواست می‌شوند:

### 1. `SEND_SMS`
- **چرا**: ارسال پیامک.
- **چه زمانی**: هنگام ارسال عادی، اجرای زمان‌بندی‌شده یا صف ارسال.

### 2. `READ_CONTACTS`
- **چرا**: انتخاب گیرنده از مخاطبین.
- **چه زمانی**: هنگام باز کردن دیالوگ مخاطبین.

### 3. `READ_PHONE_STATE`
- **چرا**: نمایش فهرست سیم‌کارت‌های فعال و برچسب‌ها (مثلاً سیم ۱ / ۲) برای انتخاب سیم ارسال.
- **چه زمانی**: هنگام باز کردن انتخاب سیم؛ در صورت رد، معمولاً فقط «سیم پیش‌فرض» در دسترس است.

### 4. `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`
- **چرا**: پایدارتر شدن اجرای پیام زمان‌بندی‌شده روی دستگاه‌های سخت‌گیر در مصرف باتری.
- **چه زمانی**: فقط در مسیر زمان‌بندی و دیالوگ اختیاری سیستم؛ می‌توانید رد کنید (رفتار بسته به دستگاه متفاوت است).

هیچ مجوزی برای تبلیغات یا آنالیتیکس استفاده نمی‌شود. بدون مخاطبین یا بدون وضعیت تلفن هم می‌توان با شماره دستی و سیم پیش‌فرض کار کرد.

## 🏗️ نحوه ساخت

### پیش‌نیازها
- Android Studio Hedgehog (2023.1.1) یا بالاتر
- JDK 17 یا بالاتر
- Android SDK 34
- Gradle 8.2+

### مراحل ساخت

1. **کلون کردن مخزن**:
   ```bash
   git clone https://github.com/mostafaafrouzi/long-sms-sender.git
   cd long-sms-sender
   ```

2. **باز کردن در Android Studio**:
   - Android Studio را باز کنید
   - "Open an Existing Project" را انتخاب کنید
   - به دایرکتوری کلون شده بروید

3. **ساخت پروژه**:
   ```bash
   ./gradlew assembleDebug
   ```
   یا از منوی Build در Android Studio استفاده کنید: Build → Build Bundle(s) / APK(s) → Build APK(s)

4. **یافتن APK**:
   - APK دیباگ: `app/build/outputs/apk/debug/app-debug.apk`
   - APK انتشار: `app/build/outputs/apk/release/app-release.apk`

### ساخت APK انتشار

برای ساخت نسخه انتشار، باید امضای دیجیتال را پیکربندی کنید:

1. ایجاد keystore (اگر ندارید):
   ```bash
   keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias
   ```

2. ایجاد `keystore.properties` در ریشه پروژه:
   ```properties
   storeFile=my-release-key.jks
   storePassword=رمز-ذخیره-شما
   keyAlias=my-key-alias
   keyPassword=رمز-کلید-شما
   ```

3. ساخت نسخه امضا شده:
   ```bash
   ./gradlew assembleRelease
   ```

## 🚀 استفاده از GitHub Release

این پروژه از **GitHub Actions** برای انتشار خودکار استفاده می‌کند:

### ایجاد یک Release

1. **ایجاد و ارسال یک تگ** (مثال برای v1.1.0):
   ```bash
   git tag -a v1.1.0 -m "نسخه 1.1.0"
   git push origin v1.1.0
   ```

2. **GitHub Actions به طور خودکار**:
   - APK انتشار را می‌سازد
   - آن را امضا می‌کند (اگر keystore پیکربندی شده باشد)
   - یک GitHub Release ایجاد می‌کند
   - APK را به release ضمیمه می‌کند

### پیکربندی امضا (توصیه می‌شود برای به‌روزرسانی روی نصب قبلی)

Secrets مورد انتظار workflow:

| نام | توضیح |
|-----|--------|
| `KEYSTORE_BASE64` | فایل `.jks` به صورت **یک خط base64** |
| `KEYSTORE_PASSWORD` | رمز فایل keystore |
| `KEY_PASSWORD` | رمز کلید |
| `KEY_ALIAS` | نام مستعار کلید |

نمونهٔ تبدیل به base64 در لینوکس:

```bash
base64 -w 0 my-release-key.jks
```

خروجی را در `KEYSTORE_BASE64` قرار دهید. **همان keystoreای کافه‌بازار / نسخه قبلی** را استفاده کنید تا به‌روزرسانی بدون حذف اپ نصب شود.

بدون این Secrets، APK خروجی build release است ولی با کلید انتشار شما یکی نیست مگر اینکه محلی امضا کنید.
